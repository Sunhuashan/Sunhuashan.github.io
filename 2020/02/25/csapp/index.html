CSAPP—第一章

  • 信息就是位+上下文

    ​ 以程序员在编辑器中创建并保存的 hello.c 文件为例，源程序实际上就是有0和1组成的序列，每个0或1为一位，每8位组成一个字节，每个字节表示一个ASCII字符。这种表示方法说明了一种思想：系统中的所有信息——包括磁盘文件、存储器中的程序、存储器中的用户数据以及网络上传送的数据，都是有一串位组成的。而区分不同数据对象的唯一方法就是读取这些数据的上下文。在不同的上下文中，一串字节序列可能被表示成不同的数据或者机器指令（这取决与这串字节序列所的系统总线）。

  • 程序被其他程序翻译成不同格式

    ​ hello.c 是由高级语言写成的文件，这种文件是人类可以读懂但计算机无法理解的。但是，为了能在计算机系统中运行 hello.c 文件，此程序必须被其他程序翻译成系统可以读懂的机器语言。这个翻译过程可以分为四个阶段，如图。执行这四个阶段的程序（预处理器、编译器、汇编器和链接器）一起构成了编译系统（compilation system）。

    预处理阶段 。预处理器程序（cpp）根据以字符”#“开头的命令，修改原始的C程序。比如，#include，就是告诉预处理器将头文件stdio中的内容添加到 hello.c 文件中，生成hello.i 文件。

    编译阶段。编译器（ccl）将 hello.i 文件通过一个汇编语言程序，将文件翻译为汇编语言的形式 保存为 hello.s。汇编语言的每条语句都确切的描述了一条低级的机器指令。汇编语言为不同的高级语言提供了一个统一的输出语言。

    汇编阶段。汇编器（as）将 hello.s 文件翻译成机器语言指令，并将这些指令打包成一种叫做可重定位目标程序（relocatable object program）的格式，并保存在文件hello.o 中。hello.o 是二进制文件，它的字节编码为机器语言而不是字符。

    链接阶段。链接器（ld）将hello.c 中调用到的库函数的预编译完成的文件与hello.o连接在一起，形成可执行目标文件hello，hello文件可以被加载到内存中，由系统执行。

  • 了解编译系统如何工作是大有益处的

    1.优化程序性能

    ​ 为了在C程序中做出好的编码选择，我们需要了解一些机器代码以及编译器将不同的C语句转换为机器代码的方式。

    • 一个switch语句是否总是比一系列的if-then-else语句高效得多？
    • 一个函数调用的开销有多大？
    • while循环比for循环更有效吗？
    • 指针引用比数组索引更有效吗？
    • 为什么将循环求和的结果放到一个本地变量中，与将其放到一个通过引用传递过来的参数中相比，运行速度要快很多呢？
    • 为什么我们只是简单的重新排列一下一个算术表达式中的括号就能让一个函数运行得更快?

    2.理解链接是出现的错误

    • 一些令人困扰的程序错误往往都是和链接器的操作有关链接器报告它无法解析一个引用，这是什么意思？
    • 静态变量和全局变量的区别是什么？
    • 如果你在不同的C文件中定义了名字相同的两个全局变量会发生什么？
    • 静态库和动态库的区别是什么？
    • 我们在命令行是哪个排列库的顺序有什么影响？
    • 为什么有些链接错误知道运行时才会出现？

  • 处理器读并解释存储在存储器中的指令

    ​ 此时 hello.c 文件已经被编译系统翻译成了可执行目标文件 hello ，并存放在磁盘上。要想在Unix系统上运行该文件，我们将它的文件名输入到 shell 应用程序中：

    unix> ./hello

    hello,world

    unix>

    ​ shell 是一个命令行解释器，它输入一个提示符，等待你输入一个命令，然后执行该命令。在此例中，shell 将加载并运行hello 程序，等待程序终止。hello 程序终止后，shell 随后输入一个提示符等待下一个命令的输入。

    1.系统的硬件组成

    ​ 为了理解hello 运行时发生了什么，我们要了解一个典型系统的硬件组成。下图为 Intel Pentium 系统产品系列的模型，其他所有的系统都有类似的硬件组成。

    ​

    总线

    总线是贯穿系统的一组电子管道，它携带着信息字节并负责在各个部件中传递。通常字节被设计成传送固定长度的字节块，被称作字（word）。字中的字节数通常是一个固定的参数，在不同系统中参数不尽相同。目前大多数机器字长有的是四个字节（32位），有的是八个字节（64位）。

    I/O 设备

    输入输出设备是系统与外界的联系通道。该示例系统有四个 I/O 设备：用户输入用到的鼠标和键盘，用户输出用到的显示器，用于长期储存数据和程序的磁盘。最初hello 程序就是存储在磁盘上。每个I/O设备都通过一个控制器或适配器与I/O 总线相连。它们的主要功能是在I/O设备与I/O总线间传递信息。

    主存

    主存是一个临时存储设备，处理器执行程序时，用来存放程序和程序处理的数据。物理上，主存是由一组动态随机存取存储器（DRAM）芯片组成的。逻辑上，主存就是一个线性的字节数组，每个字节有自己独立的编址，这些地址是从零开始的。一般的，机器指令都是有不同的数量的字节数组成的。例如，在运行Linux IA32机器上，C语言的short类型的数据存储需要2个字节，int，float 和 long类型的数据存储需要4个字节，而 double 类型需要8个字节来存储。

    处理器

    中央处理单元（CPU)，简称处理器，处理器的核心是一个字长的储存设备（或寄存器），称为程序计数器（PC）。在任何时刻，PC都指向主存中的某条指令（即，含有该条指令的地址）。从系统通电开始，到系统断电，处理器一直执行程序计数器指向的指令，在更新计数器，使其指向下一条指令。这样的操作是围绕着主存，寄存器文件和算术逻辑单元（ALU）进行的。CPU 在指令的要求下会做如下操作：

    加载：把一个字节或一个字从主存复制到寄存器，更新寄存器的内容。

    存储：把一个字节或一个字从寄存器复制到主存的某个位置，更新该位置内容。

    操作: 把两个寄存器的内容复制到ALU ，ALU 进行算术操作，将结果放到一个寄存器中，更新寄存器内容。

    跳转: 从指令本身中抽取一个字，将其复制到PC中，更新PC中的内容。

    2.运行hello程序

    • 开始时，shell等待我们输入一个命令。我们输入命令 “./hello” 后，shell程序将字符逐一读取到寄存器，然后把它存放在存储器中。

    ​

    • 当我们按下回车键时，shell 执行一系列指令加载可执行文件 hello ，利用DMA 技术将hello 文件中的代码和数据从磁盘复制到主存。
    • 处理器执行 hello 程序的 main 程序机器指令。这些指令将”hello world/n“字符串中的字节从主存复制到寄存器文件，再从寄存器文件复制到显示设备，最终显示到屏幕。

  • 高速缓存至关重要

    系统会花费大量时间把信息从一个地方挪到另一个地方：

    • 程序的机器指令最初存储在磁盘上，当程序加载时，它们被复制到主存上.
    • 当处理器运行程序时，指令又从主存复制到处理器。
    • 数据串”hello world“，最初存储在磁盘，然后复制到主存，最终从主存复制到显示设备后输出。

    ​ 显然，系统设计者的主要目标就是减少这些复制的时间成本。针对这种差异，系统设计者采用了更小，更快的存储设备，即高速缓存存储器，作为数据暂时的集结区域，用来存放近期处理器可能需要的信息。

    ​ 高速缓存存储器使用了局部性原理，即程序具有访问局部代码和数据的特点。意识到高速缓存存在的应用，程序员可以利用这一点提高程序性能。

  • 存储设备形成的层次结构

    ​ 存储器层次结构的主要思想是一层上的存储器作为低一层存储器的高速缓存。因此，寄存器文件就是L1的高速缓存，L1是L2的高速缓存，L2是L3的高速缓存，L3是主存的高速缓存，而主存又是磁盘的高速缓存。在某些具有分布式文件系统的网络系统中，本地磁盘就是存储在其他系统中磁盘上的数据的高速缓存。

    ​ 可以运用不同的高速缓存的知识来提高程序性能一样，程序员同样可以利用对整个存储器层次结构的理解来提高程序性能。

  • 操作系统管理硬件

    ​ shell 程序和 hello 程序都没有直接访问键盘、显示器、磁盘或者主存。取而代之的是，它们依靠操作系统实现与硬件之间的交互。我们可以把操作系统看做是应用程序与硬件之间的一层软件。 操作系统的两个基本功能：防止硬件被失控的应用程序滥用向应用提供简单一致的机制来控值复杂而又通常大相径庭的低级硬件设备。操作系统通过几个抽象概念来实现这两个功能。文件是对I/O设备的抽象，虚拟存储器是对主存和I/O设备的抽象，进程则是对处理器，文件和I/O设备的抽象。 进程:

    ​ 进程是操作系统对正在运行的程序的一种抽象。在一个系统上可以同时运行多个进程，而每个进程似乎都独占硬件。这是通过一个进程与另一进程的指令交错执行实现的，这种交错运行的机制叫做上下文切换。 上下文：操作系统保持跟踪进程运行所需的所有状态信息，例如，PC和寄存器文件的当前值，以及主存的内容。

    ​ 上下文切换：保存当前进程的上下文、恢复新进程的上下文。

    ​ 示例两个并发的进程：shell 进程和 hello 进程。起初，只有shell 进程在运行，等待命令行的输入。当我们输入指令后，shell 程序通过调用系统调用来执行我们的请求，系统调用将控制权转交给操作系统。操作系统保存shell 进程的上下文并创建一个新的hello 进程及其上下文，然后将控制权转交给hello 进程。hello 进程终止后，操作系统恢复 shell 进程的上下文，将控制权转交给 shell 进程，shell 进程等待下一个命令行的输入。

    ​ 实现进程等问题和细节会在后面详细介绍。

线程：

​ 现代系统中，一个进程实际上可以有多个被称为线程的执行单元组成，每个线程的运行都记录在进程的上下文中，并共享全部的代码和全局数据。多线程之间比多进程之间更容易共享数据，同时线程一般来说比进程更高效。后面会介绍多线程，并发以及如何书写线程化的程序。虚拟存储器虚拟存储器为每个进程提供了一种该进程都在独占地使用主存的假象。每个进程看到的是一致的存储器，称为虚拟地址空间。下图为Linux进程的虚拟地址空间。 Read-only code data ＆ Read/write data（统称为程序的代码和数据）

​ 对于所有的进程来说，代码是从一个固定的地址开始执行的。代码区和数据区就是直接按照可执行目标文件进行初始化的，示例中即为hello 文件。

Run-time heap （堆）

​ 代码区和数据区在进程一开始运行时就被规定好了，heap 与此不同，当调用C中的 malloc 和 free 这样的标准库函数时，heap 可以在运行时动态的拓展和收缩。

Memory mapped region for shared libraries（共享库的存储器映射，即共享库）

​ 共享库就是一块用来存放像C标准库和数学库这样的库的代码和数据的区域。

User stack（栈）

​ 用户栈位于虚拟地址空间的顶部，用户用它来实现函数调用。和 heap 一样，用户栈在进程执行期间也可以动态的拓展和收缩。我们每次调用一个函数时，栈就会增长；从一个函数返回时，栈就会收缩。

Kernel virtual memory （内核虚拟存储器，用户代码不可见）

​ 内核总是驻留在内存中 ，是操作系统的一部分。虚拟地址空间为内核保留，不允许应用读写这个区域或调用内核代码定义的函数。

虚拟存储器运作的基本思想是把一个进程虚拟存储器的内容存储在磁盘上，然后用主存作为磁盘的高速缓存。文件文件就是字节序列，仅此而已。每个I/O 设备，包括磁盘、键盘、显示器，甚至网络，都可以视为文件。系统中的所有输入输出都是通过Unix I/O 的系统调用通过读写文件来实现的。文件向应用程序提供了一个统一的视角，来看待系统中可能包含的各种各样的I/O 设备。

  • 系统之间通过网络通信

​ 现代系统通常时通过网络与多个系统相连的。从一个单独的系统来看，网络可以视作一个 I/O 设备。当系统从主存将一串字节复制到网络适配器时，数据经过网络到达另一台机器。

​ 示例：在远程主机上运行hello 程序。

  • 几个重要主题和本章总结

​ 一个很重要的观点：系统不仅仅是硬件，而是硬件与系统软件交织而成的集合体。

并发与并行

​ 计算机进步动力：我们想让计算机做的更多；我们想让计算机运行的更快。

​ 并发：一个同时具有多个活动的系统。

​ 并行：用并发使一个系统运行的更快。

​ 线程级并发：单处理器系统通过计算机在它执行的进程间快速切换的方式实现。多处理器系统通过超线程（同时多线程）的方式实现，允许一个CPU执行多个控制流。

​ 指令级并行：现代的处理器可以同时执行多条指令。

​ 单指令，多数据并行：现代处理器的硬件允许一条指令产生多个可以并行执行的操作，这种方式叫做SIMD 并行。

计算机系统中抽象的重要性

前面提到，文件是对I/O 的抽象，虚拟存储器是对程序存储器的抽象，进程是对正在运行的程序的抽象。这里添加一个新的抽象，即虚拟机（Virtual machine），它提供了对整个计算机的抽象（操作系统，处理器和程序）。

  • 小结

    • 计算机系统是由硬件和系统软件组成的，它们共同协作以运行应用程序。计算机内部的信息被表示为一组组的位，它们依据上下文有不同的解释方式。程序被其他程序翻译成不同的形式，开始时是ASCII文本，然后被编译器和链接器翻译成二进制可执行文件。
    • 处理器读取并解释存放在主存里的二进制指令。因为计算机把大量的时间用于存储器、I/O 设备和CPU寄存器之间复制数据，所以将系统中的存储设备划分成层次结构—CPU寄存器在 顶部，接着是多层的硬件高速缓存存储器、DRAM主存和磁盘存储器。在层次模型中，位于更高层的存储设备比低层的存储设备要更快，单位比特开销也更高。层次结构中较高层次存储设备可以作为较低层次设备的高速缓存。通过理解和运用这种存储层次结构的知识，程序员可以优化C程序的性能。
    • 操作系统内核是应用程序和硬件之间的媒介。它提供三个基本的抽象：1）文件是对I/O 设备的抽象；2）虚拟存储器是对主存和磁盘的抽象；3）进程是对处理器、主存和I/O设备的抽象。
    • 网络提供了计算机系统之间通信的手段。从特殊系统的角度来看，网络就是一种I/O 设备。
